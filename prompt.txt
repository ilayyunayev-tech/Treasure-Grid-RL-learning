אתה בונה משחק למידה בשם "רואים דרך ה-AI" — גרסה שמרחיבה את אותו רעיון ואותו סטאק, עם ארכיטקטורה שמאפשרת למשתמשים (ילדים) להוסיף משחקים חדשים (למשל Snake) וללמד את הסוכן מאפס דרך משקלים, כשהכל נראה ב-UI.

## סטאק (חובה — זהה לפרויקט המקורי)
- דפדפן בלבד: HTML + CSS + JavaScript, **מודולי ES** (`type="module"`), בלי React/Vue.
- הרצה: שרת סטטי (`npx serve .` או Live Server).
- מבנה קבצים דומה למקור:
  - `index.html`
  - `src/config.js` — קבועים (רשת, α, ε, γ, תגיות)
  - `src/features.js` — פיצ'רים בינאריים/דיסקרטיים ∈ {-1,0,1} (או מוגדרים פר-משחק)
  - `src/ai.js` — ציון ליניארי לכל פעולה + ε-greedy
  - `src/learning.js` — עדכון משקלים (TD: פרס + γ·max(Q') − Q)
  - `src/game.js` — **מנוע משחק אחד לפחות מובנה** (Treasure Grid) + **ממשק הרחבה**
  - `src/render.js` — ציור לוח/מצב
  - `src/input.js` — מקלדת + כפתורים
  - `src/ui.js` — פאנלים, משקלים, יומן למידה, חישוב AI גלוי
  - `src/main.js` — לולאת תורים + delay קבוע בין תורים
  - `src/errorToasts.js` — שגיאות עם מיקום בקוד
- README בעברית עם הוראות הרצה וטבלת קבצים.

## משחק ברירת מחדל (חייב להישאר זהה ברוחו)
- לוח grid, שחקן אנושי + סוכן AI, תנועה בו-זמנית, אוצרות ומלכודות, מצב תחרות (race) אופציונלי.
- פיצ'רים לדוגמה (5): קרבה לאוצר, התרחקות ממלכודת, מוביל לאוצר מול יריב, תא חדש, תא ריק.
- UI: סליידרים/שדות למשקלי AI, החל / ברירת מחדל / אפס, "משחק חדש + AI מאפס".
- פאנל "חישוב AI" — לכל כיוון: ציון כולל + פירוט תרומת כל פיצ'ר (משקל × ערך).
- פאנל "יומן למידה" — אחרי כל תור: עדכון משקלים/טעות TD, ניתן לכיווץ.
- ε יורד עם הזמן; למידה ניתנת להפעלה/כיבוי.

## דרישת הרחבה (הליבה של המשימה)
בנה **שכבת "Game Definition"** או **Plugin API** כך שאפשר יהיה (בלי לשבור את Treasure Grid):

1. **משחק = אובייקט/מודול** שמגדיר:
   - מימדים (grid או גרף צעדים ל-Snake)
   - רשימת פעולות (4 כיוונים או Up/Down/Left/Right בהתאם)
   - `getStateKey(state)` — מפתח למילון Q/משקלים
   - `getFeatures(state, action)` — וקטור פיצ'רים לאותה פעולה
   - `step(state, humanAction, aiAction)` — מעבר מצב, פרסים, סוף משחק
   - `render(canvas/dom, state)` או callback ל-render
   - תוויות פיצ'רים ותוויות פעולות — ל-UI

2. **מקום לחוקים חדשים**: קובץ או תיקייה `src/games/` עם:
   - `treasureGrid.js` (ברירת מחדל)
   - דוגמה מינימלית `snakeSolo.js` או תבנית ריקה — סנייק שבו ה-AI לומד לאכול / לא להתנגש (גם אם ב-MVP רק שלד + פיצ'רים בסיסיים), כדי להראות שאפשר להחליף משחק.
   - מסמך `docs/extending-games.md` — איך ילד/מורה מוסיף משחק חדש (רשימת פונקציות חובה).

3. **בחירת משחק ב-UI**: dropdown או טאבים — "Treasure Grid" | "Snake (דמו)" | עתידי. מעבר משחק מאפס מצב למידה או שומר פר-משחק (הגדר במפורש ב-README).

4. **למידה מאפס**: בכל משחק חייב להיות מצב "כל המשקלים 0" + ε > 0 — הילד רואה ביומן ובחישוב AI איך התנהגות נוצרת רק מחוקי המשחק + חקר.

5. **הכל ב-UI**: אותו שקיפות — אין "קופסה שחורה". כל משחק חדש חייב לחשוף את אותם פאנלים (או גרסה מותאמת עם אותה פילוסופיה: פיצ'רים + משקלים + עדכון אחרי צעד).

## עקרונות חינוכיים (אל תדרוס)
- לא להסביר ML במילים ארוכות במשחק; לתת גילוי דרך משקלים וכשל.
- Treasure Grid נשאר חוויית הליבה; Snake/אחרים הם הרחבה או דמו.
