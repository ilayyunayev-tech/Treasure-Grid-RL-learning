# 🎮 המשחק במשפט אחד

**אתה ו-AI רצים על לוח לאוצרות**  
כל תור:

1. אתה בוחר כיוון  
2. ה-AI מראה לך את החשבון שהוא עושה  
3. שניכם זורקים ורואים מה קרה  
4. ה-AI לומד מהטעות שלו (ואתה רואה *איך*)

**המטרה:** להבין איך המשקלים שלו עובדים — ולתכנן מהלך שהוא *לא יצפה*

# 🔢 State — המידע שה-AI רואה (גרסה ראשונה / אינטואיטיבית)

```
1. איפה אני (x, y)
2. איפה האוצר הקרוב (dx, dy)
3. איפה האויב (dx, dy)
4. האם יש מלכודת קרובה? (0/1)
5. האם אני קרוב יותר מהאויב? (0/1)
```

**למה חשוב:**  
ככל שיש יותר state → יותר משקלים → יותר חשבון  
אבל: יותר מ-5 פיצ'רים = הילד מפסיק לעקוב

👉 **5 זה המקסימום**

# 🧮 Features — החישוב שה-AI עושה (גרסה מודגמת)

כל feature מקבל ערך **-1 / 0 / +1**  
(לא יותר! אחרת זה מתפוצץ)

### דוגמה לתור אחד:

```
Feature 1: קרוב לאוצר?
  dx_treasure = -2 → normalize → -1
  
Feature 2: רחוק ממלכודת?
  no trap nearby → 0
  
Feature 3: קרוב יותר מהיריב?
  yes → +1
```

**עכשיו הכפל:**

```
משקל 1 (אוצר):     2.5 × (-1) = -2.5
משקל 2 (מלכודת):   1.0 × (0)  =  0
משקל 3 (יתרון):    1.2 × (+1) = +1.2

סכום = -1.3 ← זה הציון של "שמאלה"
```

**הילד רואה:**

```
⬅ LEFT
─────────────
קרוב לאוצר:    -2.5 ⚠️
רחוק ממלכודת:   0
יתרון על יריב: +1.2 ✓
─────────────
סה"כ: -1.3 ❌ (לא שווה)
```

# 🎯 Rewards — מתי ה-AI מקבל נקודות (גרסה ראשונה)

| אירוע | נקודות | למה? |
|-------|---------|------|
| אספתי אוצר | +10 | זה המטרה |
| נפלתי במלכודת | -10 | זה רע |
| זזתי למשבצת חדשה | +1 | עידוד לחקור |
| עמדתי במקום | 0 | לא קרה כלום |
| ניצחתי את השחקן | +5 | בונוס |

**הערה:** בגרסה הסופית המתוקנת (להלן) — **Exploration reward הוסר**; רק אוצרות, מלכודות וקונפליקט אוצר משותף.

# 🔄 Learning — איך ה-AI משתפר

אחרי כל תור:

```
1. ה-AI רואה כמה נקודות קיבל (reward)
2. משווה: "מה חשבתי שיקרה" vs "מה באמת קרה"
3. מעדכן משקלים בכיוון של הטעות
```

### דוגמה שהילד רואה:

```
💭 AI חשב: ימינה = +3
🎯 קיבל: +10 (אוצר!)
📈 טעות: +7

עדכון משקלים:
• "קרוב לאוצר": 2.0 → 2.3 ⬆
  (כי זה עזר!)
```

**הנוסחה (בגרסה הסופית):**  
`weight ← weight + α × error × feature` עם **α = 0.03**

# 🔧 טכני (בלי קוד) — מבנה קבצים

```
/src
  config.js      ← קבועים (grid size, learning rate, ε)
  main.js        ← bootstrap
  game.js        ← state, rules, game loop
  ai.js          ← decision (ε-greedy + score calculation)
  learning.js    ← weight updates
  features.js    ← feature calculations (normalization)
  render.js      ← drawing only
  input.js       ← user input
  ui.js          ← explanation panels
```

**למה חשוב?**  
→ אם תערבב game+ai = תאבד שליטה  
→ אם תערבב learning+features → למידה שגויה

# Game Loop (הסדר הנכון)

```
1. INPUT — שחקן בוחר; AI מחשב ציונים
2. DECISION — ε-greedy; הצגת חישוב
3. EXECUTION — שניהם זזים; התנגשויות
4. ENVIRONMENT — אוצרות/מלכודות; קונפליקטים
5. REWARD — ציון לכל שחקן; "מה קרה"
6. LEARNING (AI בלבד) — error; עדכון weights; "מה למדתי"
7. STATE UPDATE
8. RENDER
```

**למה הסדר חשוב?**  
→ אם תלמד לפני reward = למידה שגויה  
→ אם תציור לפני state = חוסר סנכרון

# 📐 State — המידע שה-AI רואה (בדיוק 5 — גרסה סופית)

```
1. מרחק לאוצר הקרוב (dx, dy)
2. מרחק לאויב (dx, dy)
3. האם יש מלכודת ב-1 צעד ממני?
4. האם אני קרוב יותר מהאויב לאוצר?
5. האם התא שאני רוצה ללכת אליו ריק?
```

**למה דווקא 5?**

# 🧮 Features — החישוב המדויק (גרסה סופית)

## כל feature = פונקציה דטרמיניסטית, תמיד ∈ {-1, 0, +1}

### Feature 1: מתקרב לאוצר?

```
before = מרחק_נוכחי_לאוצר
after  = מרחק_אחרי_תזוזה
feature = sign(before - after)
→ +1 אם מתקרב, 0 אם אותו מרחק, -1 אם מתרחק
```

### Feature 2: מתרחק ממלכודת?

```
if אין_מלכודת_קרובה: feature = 0
else:
  before = מרחק_למלכודת
  after  = מרחק_אחרי_תזוזה
  feature = sign(after - before)
```

### Feature 3: שומר על יתרון?

```
before = האם_הייתי_קרוב_יותר
after  = האם_אהיה_קרוב_יותר_אחרי_תזוזה
feature = +1 אם after = true, אחרת 0
```

### Feature 4: תא חדש?

```
feature = +1 אם לא_הייתי_שם, אחרת 0
```

### Feature 5: תא ריק?

```
feature = +1 אם ריק, -1 אם תפוס
```

# 🎯 Decision — איך ה-AI בוחר

## שלב 1: חישוב ציון לכל כיוון

```
score(dir) = w₁×f₁ + w₂×f₂ + w₃×f₃ + w₄×f₄ + w₅×f₅
```

**הילד רואה את זה פתוח** (דוגמה ל-UP):

```
קרוב לאוצר:     +1 × 2.3 = +2.3 ✓
רחוק ממלכודת:     0 × 1.5 =  0
שומר יתרון:      +1 × 0.8 = +0.8 ✓
תא חדש:          +1 × 0.2 = +0.2
תא ריק:          +1 × 0.5 = +0.5 ✓
                         ─────
                 סה"כ: +3.8
```

## שלב 2: ε-greedy

```
80% → בחר כיוון עם score מקסימלי
20% → בחר כיוון אקראי
```

**הילד רואה** הגרלה (מספר, האם נפל ב-80% או ב-20%) והסבר.

# 🏆 Rewards — הציונים האמיתיים (גרסה סופית)

| אירוע | ציון |
|-------|------|
| אספתי אוצר | +10 |
| נפלתי במלכודת | -10 |
| ניצחתי (אוצר משותף) | +3 |
| הפסדתי (אוצר משותף) | -3 |
| כלום | 0 |

**למה לא "+1 לתא חדש"?**  
→ גורם ל-AI לרוץ בלי כיוון; שובר למידה (ה-feature "תא חדש" נשאר במודל, אבל אין reward עליו)

# 🔄 Learning — גרסה סופית

```
w_new = w_old + α × error × feature
α = 0.03
error = reward - predicted_value
```

(predicted_value = הציון של הכיוון שנבחר לפני התור)
